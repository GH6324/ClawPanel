# ClawPanel v5.0.2 — 面板一键自检更新 · OpenClaw 版本识别增强

**发布日期**: 2026-02-25

## ✨ 新功能

### 🔄 ClawPanel 面板一键自检更新（国内加速）
- 新增面板自检更新功能，基于北京服务器（39.102.53.188）加速，解决中国大陆用户访问 GitHub 不稳定导致的更新困难问题
- 系统配置 → 版本管理页面新增「ClawPanel 面板更新」卡片：
  - 显示当前面板版本号
  - 「检查更新」按钮，从国内加速服务器获取最新版本信息
  - 有新版本时展示版本号、发布时间、更新日志
  - 「立即更新」按钮，一键下载、SHA256 校验、替换程序、自动重启
  - 实时进度条和更新日志展示
- 更新完成后自动重启面板，前端自动刷新页面
- 更新成功后首次登录弹出更新弹窗，展示版本号及更新内容
- 全程无需用户手动访问 GitHub、下载文件、替换程序
- 更新过程自动备份旧程序（`.bak` 文件），SHA256 校验防损坏
- API：
  - `GET /api/panel/version` — 获取面板版本
  - `GET /api/panel/check-update` — 检查面板更新
  - `POST /api/panel/do-update` — 执行面板更新
  - `GET /api/panel/update-progress` — 获取更新进度
  - `GET /api/panel/update-popup` — 获取更新弹窗
  - `POST /api/panel/update-popup/shown` — 标记弹窗已显示

## 🐛 Bug 修复

### 🔍 OpenClaw 版本识别增强（#15）
- **修复 OpenClaw 版本识别不全**：之前仅检查配置文件 `meta.lastTouchedVersion` 和 `openclaw --version`，很多安装方式无法识别
- 现在按以下优先级全面检测：
  1. `openclaw --version` CLI（PATH 中的全局安装）
  2. `openclaw.json` 配置文件中的 `meta.lastTouchedVersion`
  3. npm 全局安装目录的 `package.json`
  4. 常见二进制路径扫描（`/usr/local/bin`、`~/.local/bin`、`~/.npm-global/bin`）
  5. 常见源码安装目录的 `package.json`（`~/openclaw`、`/opt/openclaw`）
  6. Docker 容器内执行 `openclaw --version`
  7. systemd 服务文件解析 `ExecStart` 路径
  8. 兜底：配置文件存在但无法提取版本号时返回 "installed"
- 状态总览页和软件环境页共用同一检测逻辑，结果一致
